System Properties and the .jmaxrc File

jMax defines the following system properties:
jmaxConnection
type of the connection between the jMax client and server:

client . . . automatic start up of server and connection via socket (on free port)
socket . . . socket connection to independently started server (requires port number)
local . . . local connection (e.g. pipe)

jmaxPort
port number for socket connection

jmaxHost
name of the host running the jMax server

jmaxArch
architecture of host running the jMax server

jmaxMode
can be opt or debug (better do not touch)

jmaxServerName
name of server executable
(this can be used to run an alternative version of the jMax server)

jmaxAudioBuffer
default audio buffer size declared to jMax at startup
(see Audio Configuration down)

jmaxSampleRate
default sample rate declared to jMax at startup
(see Audio Configuration down)

jmaxProfiling
starts up with profiling pseudo audio device installed when set to on
(see Audio Configuration down)

jmaxMidiPort
default MIDI port opend at startup
(see MIDI Configuration down)

jmaxSplashScreen
supresses jMax startup splash screen when set to hide

jmaxPkgPath
(not yet implemented)

The installation should provide a default value for each of these system properties, which should permit you start up jMax without any comand line arguments or further declarations (refer to Installation Instructions).

To overwrite these defauts you can either startup jMax with a command line arguments or do declare your own settings
