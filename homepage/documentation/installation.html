Installing the jMax distribution

Ones you have got the packaged jMax distribution files (e.g. by downloading them from the jMax home page) you want to execute the following steps:
  1. Uncompress the package:

    gunzip <jMax distrib>.tar.gz

  2. If it is not already there copy the resulting <jMax distrib>.tar to the directory where you want to create the root directory of this installation and remove (if necessary) older installations of jMax in this directory.

    In order to benefit from some default settings of the distribution we recommend here the directory /usr/local.

  3. Now, unpack the distribution:

    tar -xf <jMax distrib>.tar

    This will create the jMax root directory with all files inside.

  4. Have a look to the README file.

Java Installation

The current version of jMax requires an installation of the Java Execution Environment (JRE) version 1.1.5 or higher, which is aswell distributed together with the Java Development Kit (JDK).

How to install the JRE on your mashine will be better explained there, where you get it.

Please refer to the links page of the jMax home page for further information.

Configuration files

In the current distribution the configuration for an installation depends on three files in the config directory of the distribution:
defaults.tcl
installation default settings of the system properties
(overwritten by user settings and commandline arguments)

packages.tcl
packages loaded by default for this installation
(may depend on system properties)

audio.cfg
default audio configuration at startup
(may depend on system properties)

midi.cfg
default MIDI configuration at startup
(may depend on system properties)

These files should assure that a user can startup jMax anytime running a minimum configuration without further declarations.
They should be modified to match your installation (at least edit defaults.tcl to set the right server architecture).
Typically an installation is multiplatform and shared by multiple machines of a local network.

The jMax user can give his own values for the system properties in his user configuration file ~/.jmaxrc.
You find an example of a user configuration file the config directory of the distribution as examples.jmaxrc.

Please refer to Configuration and Environment Setup for a detailed description of the system properties and the user configuration file.

Running jMax

If your installation matches the following points

  1. the jMax distribution is installed in /usr/local
  2. a Java environment is installed in /usr/java

just run the jmax script in the bin directory of the distribution.
If not fit this script for your installation.

The system properties given by a default in defaults.tcl or in the user configuration file. can be overwritten by command line arguments to jmax:

jmax -<system property name> <system property value>

Additional system properties can be defined by using the same syntax (see Configuration and Environment Setup).
