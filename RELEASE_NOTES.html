jMax release notes for version 2.3.9

Fixed bugs

The following bugs have been fixed:
  • #110: Deleting an outlet in a template crashes jMax
  • #111: + doesn't work in expressions
  • #115: fast graphic updates with dsp on
  • #116: typing in objects
  • #117: downsampled line~
  • #118: inputs don't match crash
  • #123: float and int box don't keep size on copy/paste and save/open
For bugs reported via the bug report system (http://www.ircam.fr/bugs, please refer to this page for information.

Changed features

  • the jmax ``-install'' option has been removed. The root directory (i.e. the directory where jMax takes the Java classes, the server, the libraries, etc) is given by the directory from which the script jmax is started. As a consequence, to start:
    • the standard installation (in /usr/local/max): start /usr/local/max/bin/jmax
    • the snapshot installation (usually in /usr/local/max-snapshot): start /usr/local/max-snapshot/bin/jmax

New features

  • the system property "FastFileBox" is now exported as a Tcl variable, so that you can set it in your .jmaxrc file: 
    set jmaxFastFileBox "true"
    as well as by the command line: 
    jmax -jmaxFastFileBox
    By default, this variable is set to "false".
  • 24 bits audio I/O
  • improved patch opening time up to 40 %
  • detection of ADAT input not connected. Instead of FTS being blocked, an error message "No Carrier On Audio Input" is issued.
