jMax release notes for version 2.3.4

Fixed bugs

The following bugs have been fixed:
  • 101 (grave): explode with record and next message
  • 100 (normal): First paint of a patcher window
  • 97 (Look and Feel): Run mode should be named lock mode
  • (non-reported): copy/cut/paste problems
Please refer to the bug report system Web page (http://www.ircam.fr/bugs), for more information on bugs.

New features

  • jmax option -jmaxFastFileBox will get a faster FileBox, *but* there will be no filters and no icons. Thru the net, the difference is highly visible.
    To use it, either add it to your alias (i.e. ``alias jmaxs /usr/local/max-snapshot/bin/jmax -install snapshot -jmaxFastFileBox'') or pass it as argument to the script jmax or to the alias.

Known bugs

  • 96: Connections, objects and user-interface objects precedence
    Clicking on an object that overlaps another objects alters the objects precedence in a strange way.
  • 105: Etirement des objets Interface Graphique
    If a selection is present, resizing an object does not resize the selection, but resizes only the selected object.

Outstanding bugs and features

  • 103: explode (It's not possible to copy cut and paste in explode window.)
    It is not yet possible to copy, cut and paste in an explode. The menu was there and enabled by error.
