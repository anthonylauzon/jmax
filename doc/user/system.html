Disclamier

The current version of jMax use TCL as its scripting language, and the feature described in this document is currently based on TCL; future version of jMax may use a different scripting language, so the features described here will change, will be implemented with a different language and/or syntax, or will simply vanish.

Introduction

The system configuration is based on system parameters that are configuration options that can be set either by setting a TCL variable or passing a command line argument.
These options allow the setting of the host where the server will run, the architecture of the host, the kind of connections and a number of miscellaneous settings and options.

System Configuration Parameters

Not all the options of the jmax script are documented here; to see script specific options, run the jmax -help command.

jmaxArch

Set the architecture where the server will run; used to choose the proper executable of the server; consult the release notes in order to know the architecture supported in this release; in this release, this comand is mandatory; if you don't specify the good architecture, jMax will not run.

jmaxConnection

Choose the kind of connection between the Java client and the server; Supported values are currently udp for an UDP based connection, and client for a TCP based connection; UDP will give a better real time response for controllers; use TCP only if you network is complex and you need the reialability of a TCP connection.
Default to udp.

jmaxHost

Specify the host where to run the server; value can be either local to specify the host where the client run, or a complete canonical internet name for an other host, like mymachine.mycompany.com; beware that jMax currently use the rsh Unix command to run the server remotely, so in order to work you must set your .rhosts file correctly; see the rsh and rhosts man page for details.
Defaults to local.

jmaxSplashScreen

If this value is hide the splash screen will not be shown during startup.
Default to show.

jmaxFastFileBox

If the value is true, the file selection box used by jMax will not shown specific icons for known file types, and will always show all the files in a directory, and not only the files of a known file type, so it's use will be substantially faster.
Depending on your installation, the default speed can be ok, or you may need to specify this option.

jmaxDefaultFont

Specify the default font used in the patch editor.
Default to SansSerif.
