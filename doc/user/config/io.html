Disclamier

The current version of jMax use TCL as its scripting language, and the feature described in this document is currently based on TCL; future version of jMax may use a different scripting language, so the features described here will change, will be implemented with a different language and/or syntax, or will simply vanish.

Introduction

Since a patch can be used on different platforms, platform specific information should not be included in the patches themselves, but specified in site, user or project configuration files.

For each host type (see section host parameters) supported by jMax a set of i/o devices is defined, which can be used with these commands. Refer to the index of supported host types to see which audio devices are provided for each of them.

Audio Configuration

The following set of commands handles the audio configuration. The first two of them are used to set the audio configuration parameters and should be used before the others.

jmaxSetSampleRate sr

Set the jMax global sampling rate.
(Note: For now, the user himself must keep the sampling rate given for jMax consistent with the settings of the physical device using the utillities provided by the operating system, like the SGI audiopanel application.)

jmaxSetAudioBuffer buf

Set the length of the audio buffer used, i.e. the length, in samples, of the output FIFO used to compensate the variation of load in jMax. Typical values are between 256 and 1024, can be more or less depending on the application.
Lower values cause lower i/o latencies, while higher values reduce possible audio clicks caused by system overloads.

openDefaultAudioIn device

Open the audio input device device as the default audio device, i.e. the device used by the adc~ object.

openAudioIn name device

Open a input device named name connected to the objects in~ which have name as first argument (allowing the use of multiple input devices - one for each object).

resetAudioIn

Reset the effects of the previous commands, i.e. close all the input audio devices, and delete the registered names (usefull in projects files or other situation where a clean input configuration must be installed from scratch).

openDefaultAudioOut device

Open the audio output device device as the default audio device, i.e. the device used by the dac~ objects.

openAudioOut name device

Open an output device named name connected to the objects out~ that have name as first argument (allowing the use of multiple output devices - one for each object).

resetAudioOut

Reset the effects of the previous commands, i.e. close all the output audio devices, and delete the registered names (usefull in projects files or other situation where a clean output configuration must be installed from scratch).

MIDI Configuration

These commands handle the MIDI configuration:

openDefaultMidi device

Open the MIDI device device as the default MIDI port, i.e. the device used by all the MIDI objects without a port number argument.

openMidi num device

Open the MIDI device device as the MIDI port installed on port number num i.e. the device used by all the MIDI objects with a port number argument of num. A device installed with port number of 0 will be used by all the MIDI related object without a MIDI port number.

resetMidi

Reset the effects of the previous commands, i.e. close all the MIDI devices, and delete the registered port number (usefull in projects files or other situation where a clean MIDI configuration must be installed from scratch).
