With the term configuration, we cover all the different kind of configuration problems we find in programming with jMax, from configuring and choosing the system where jMax run, to which particular I/O device we use with our patches, and how this device is referred in the patch, to how we organize complex applications or libraries, and how we keep the dependencies between them; these subjects are covered in the four configuration sections above.
