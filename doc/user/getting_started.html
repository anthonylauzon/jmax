Running jMax

jMax can be run by executing the shell script <INSTALLATION_DIRECTORY>/bin/jmax. This script accepts a set of options (see jmax -help for description) that are either interpreted by the script or passed as command line arguments to the Java application as system properties.

System properties

jMax uses a number of properties to parametrize the system. A property is a named variable with a default value, that can be set by a Tcl script or by command-line arguments, and that can be accessed from Java code.

The system properties given by a default in defaults.tcl or in the user configuration file. can be overwritten by command line arguments to jmax script:

    -jmax<system property name> <system property value>

Additional system properties can be defined by using the same syntax (see Configuration and Environment Setup).

.jmaxrc file

The .jmaxrc file is a Tcl script that is loaded at jMax startup. It can contain any Tcl code, though the following is probably of main interest:

  • set system properties to customize your environmnet
  • declare a number of hooks, Tcl code that will be executed at precise points during the initialization process. In particular, the start hook is executed after server start and is especially usefull for configuration purpose.

A set of template .jmaxrc files can be found in the distribution.

As an example, the following file can be used for an Octane configuration, using the ADAT audio I/O: 

# --- .jmaxrc
set jmaxArch  origin
when start {
jmaxSetSampleRate 44100
jmaxSetAudioBuffer 512
ucs open device in~ default as SgiALIn ALdevice ADATIn channels 8 fifo_size [jmaxGetAudioBuffer]
ucs open device out~ default as SgiALOut ALdevice ADATOut channels 8 fifo_size [jmaxGetAudioBuffer]
}
# --- end of .jmaxrc
