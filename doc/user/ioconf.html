Disclamier

The current version of jMax use TCL as its scripting language, and the feature described in this document is currently based on TCL; future version of jMax may use a different scripting language, so the features described here will change, will be implemented with a different language and/or syntax, or will simply vanish.

Introduction

Make an I/O configuration means two things: first, choosing and opening the platform specific I/O devices that will be used, second, to associate these devices with the objects in jMax patches; since a patch can be moved between different platform, platform specif information should not be included in the patches themselves, but specified in site, user or project configuration files, by means of a set of TCL commands.

Two commands allow the control of global parameters, the main sampling rate and the audio buffer size.

I/O Configuration Commands

The following set of commands is subject to change in future releases.

jmaxSetSampleRate sr

Set the jMax main sampling rate; note that for this release, on Sgi machines the sampling rate must also be set by hand using the audiopanel for all the devices used.

jmaxSetAudioBuffer buf

Set the length of the audio buffer used, i.e. the length, in samples, of the output fifo used to compensate the variation of load in jMax; typical values are between 256 and 1024, can be more or less depending on the application.

openDefaultAudioIn device ch

Open the audio input device device as the default audio device, i.e. the device used by the adc~ objects; the device meaning is platform dependent; for the Sgi version, device is the name of the input device as it appear in the audio panel (without the blanks) (see the man page for audio keyword); depending on the workstation model and on the installed options, valid names are AnalogIn, AESIn, AdatIn, or RADx.AdatIn and RADx.AESIn for the RAD card x. The ch argument must be the number of channels supported by the interface, usually 2 or 8.

openAudioIn name device ch

As the previous command, but open an input named device connected to the objects in~ that have name as first argument; these objects are equivalent to the adc~ object, but allow the use of multiple input device, one for object.

openNamedDefaultAudioIn name device ch

Like the previous two combined; i.e. open a named input device that can be used with the in~ object, and set this device also as the device used by the adc~ object.

resetAudioIn

Reset the effects of the previous commands, i.e. close all the input audio devices, and delete the register names. Usefull in projects files or other situation where a clean input configuration must be installed from scratch.

openDefaultAudioOut device ch

Open the audio output device device as the default audio device, i.e. the device used by the dac~ objects; the device meaning is platform dependent; for the Sgi version, device is the name of the output device as it appear in the audio panel (without the blanks) (see the man page for audio keyword); depending on the workstation model and on the installed options, valid names are AnalogOut, AESOut, AdatOut, or RADx.AdatOut and RADx.AESOut for the RAD card x. The ch argument must be the number of channels supported by the interface, usually 2 or 8.

openAudioOut name device ch

As the previous command, but open an output named device connected to the objects out~ that have name as first argument; these objects are equivalent to the dac~ object, but allow the use of multiple output device, one for object.

openNamedDefaultAudioOut name device ch

Like the previous two combined; i.e. open a named output device that can be used with the out~ object, and set this device also as the device used by the dac~ object.

resetAudioOut

Reset the effects of the previous commands, i.e. close all the output audio devices, and delete the register names. Usefull in projects files or other situation where a clean output configuration must be installed from scratch.

openDefaultMidi port

Open the MIDI port as the default MIDI device, i.e. the device used by all the MIDI objects without a port number argument; the port meaning is platform dependent; for the Sgi version, device is the name of the midi port as given in the startmidi command (see the its man page for details on MIDI on IRIX).
Currently the default MIDI device is the port installed with port number 0, so this comand is equivalent to the next one with first argument 0.

openMidi num port

Open the MIDI port as the MIDI device installed on port number num i.e. the device used by all the MIDI objects with a port number argument of num; the port meaning is platform dependent; for the Sgi version, device is the name of the midi port as given in the startmidi command (see the its man page for details on MIDI on IRIX).
A device installed with port number of 0 will be used by all the MIDI related object without a MIDI port number.

resetMidi

Reset the effects of the previous commands, i.e. close all the MIDI devices, and delete the registered port number. Usefull in projects files or other situation where a clean MIDI configuration must be installed from scratch.
