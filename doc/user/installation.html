Installing the jMax distribution

jMax is distributed as a tar gzip'ed file. After downloading the jMax distribution file (for instance jmax.free.linuxpc.2.3.12.tar.gz), you will need to execute the following:
  1. untar the package: 
    tar xvzf DISTRIB-FILE
    This will create a directory max-<VERSION NUMBER>. To keep environment unchanged with new releases of jMax, you can make a symbolic link to it, as : 
    ln -s max-2.3.12 max
    Note: be carefull to untar the files with the correct access permissions.

  2. execute as root the script INSTALL.sh in jMax root directory. This script will change the access permissions of jMax server executables to allow it to switch to real-time scheduling policy and priority.

Java Installation

The current version of jMax requires:
  1. an installation of the Java Runtime Environment (JRE) or the Java Development Kit (JDK) version 1.1.5 or higher.
  2. JavaTM Foundation Classes (JFC) Swing 1.1 Beta 2

NOTE: jMax requires Swing 1.1 Beta 2. It is known not to work with Swing 1.1 Beta 1 and Swing 1.1 Beta 3 !

How to get the JRE or JDK and how to install it on your machine will be better explained there:

  • JDK: http://java.sun.com/products/jdk/1.1/index.html
  • Silicon Graphics Java Development Environment: http://www.sgi.com/Products/Evaluation/#jdk

JavaTM Foundation Classes (JFC) Swing 1.1 Beta 2 can be found at:
http://developer.java.sun.com/developer/earlyAccess/jfc/index.html
This site requires registration (which is free...).

Configuration files

In the current distribution, the configuration for an installation depends on the following files in the config directory:

defaults.tcl
installation default settings of the system properties
(overwritten by user settings and command line arguments)

packages.tcl
packages loaded by default for this installation

audio.cfg
default audio configuration at startup

midi.cfg
default MIDI configuration at startup

These files should assure that a user can startup jMax anytime running a minimum configuration without further declarations. They should be modified to match your installation (at least edit defaults.tcl to set the right server architecture).

The jMax user can give his own values for the system properties in his user configuration file ~/.jmaxrc. You find an example of a user configuration file the config directory of the distribution as examples.jmaxrc.
