Installing the jMax distribution

jMax is distributed as a tar gzip'ed file. After downloading the jMax distribution file (for instance jmax.free.linuxpc.2.3.12.tar.gz), you will need to execute the following:
  1. Untar the package in the directory where you want to install jMax
    (typically /usr/local):

      tar xvzf <distribution file>
      tar xvzf jmax.free.linuxpc.2.3.12.tar.gz

    This will create the jMax version directory as  max- <version number>.
    To keep the environment unchanged with new releases of jMax, you can make a symbolic link from the version directory to a constant jMax root directory, like:

      ln -s <version directory> <jMax root directory>
      ln -s /usr/local/max-2.3.12 /usr/local/max

    (Note: Be carefull to untar the files with the correct access permissions.)

  2. Execute the installation script in the jMax root directory with superuser permissions (as root):

      INSTALL.sh

    This script will change the access permissions of jMax server executables to allow it to switch to real-time scheduling policy and priority.

Java Installation

The current version of jMax requires:
  1. An installation of the Java Runtime Environment (JRE) or
    the Java Development Kit (JDK) version 1.1.6 or higher
  2. Java Foundation Classes (JFC) Swing 1.1

Refer to the following site for Java download and installation:

  • JDK for SGI Systems or Sun's JDK and JRE distributions for multiple platforms

NOTE: jMax requires Swing 1.1 It will not to work with Swing 1.1 Beta x !

Starting with release 2.3.12, Swing 1.1 classes are distributed with jMax, and it is not longer required to download separately Swing distribution.

The jmax script uses the jMax Swing distribution, and setting CLASSPATH is no longer required. However, if CLASSPATH is set and contains swing, the jmax script uses it in place of jMax Swing distribution.

To get full Swing distribution, refer to the following site:

  • JavaTM Foundation Classes (JFC) Swing 1.1
    (this site at Sun requires a free registration)

The path of the Java executable must be added to the PATH environment variable in order to startup the Java environment by the jMax script. This can be done for example by adding the following line to the .cshrc file in the user home directory:
      setenv PATH ${PATH}: <Java directory>
for instance: setenv PATH "${PATH}:/usr/java/bin"
