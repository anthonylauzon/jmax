Java

To run jMax, you need a Java virtual machine, which means installing on your machine either:

  • a Java Runtime Environment or JRE, which contains a virtual machine and the necessary classes libraries
  • a Java Development Kit or JDK, which contains the runtime environment plus the development tools (Java compiler, debugger,...)

If you plan to recompile jMax or develop your own Java objects, you must install the Java Development Kit

The Java Runtime Environment or Development Kit can be found at the following places:

  • on Linux:
    • Sun's JRE: http://java.sun.com/j2se/1.3/jre/download-linux.html
    • Sun's JDK: http://java.sun.com/j2se/1.3/download-linux.html
    • Blackdown's JRE and JDK: http://www.blackdown.org/java-linux/mirrors.html
    • IBM's JRE and JDK: http://www-106.ibm.com/developerworks/java/jdk/linux130/?dwzone=java
  • on SGI: http://www.sgi.com/products/evaluation/6.x_java2_1.3/

On Linux, we recommand IBM's Java virtual machine.

Once downloaded the Java virtual machine of your choice, follow its installation instruction for updating the environment variables.

ALSA

On Linux, jMax uses either the OSS audio drivers or the ALSA audio drivers.

We strongly recommand that you use the ALSA drivers which provide far better performance and support.

To download and install ALSA drivers, visit http://www.alsa-project.org.

Installation directories

jMax installation uses the following directories:

  • /usr/bin for the jmax script
  • /usr/lib/jmax for the root directory (containing Java classes libraries, server executable, configuration scripts, standard packages, ...)

jMax documentation is installed in the following directories:

  • /usr/webdocs/jMax on SGI platform
  • /usr/doc/jMax-$(version) on Linux platform
