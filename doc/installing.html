This documentation does not apply if you installed jMax from a package (rpm for example), the installer will take care of all installation settings. If you compiled jMax from a source distribution, you can either install it by 1) building a package (rpm on Linux) or 2) use the makefile install target. The first method is preferable.

Installation

jMax can be installed under make control, by using the install target and specifying the architecture by the make variable ARCH, as in: 

make install ARCH=i686-linux

This will copy the necessary files to the standard installation directories.

Installation directories

jMax installation uses the following directories:

  • /usr/bin for the jmax script
  • /usr/lib/jmax for the root directory (containing Java classes libraries, server executable, configuration scripts, standard packages, ...)
  • /usr/lib/jmax for the root directory (containing Java classes libraries, server executable, configuration scripts, standard packages, ...)

jMax documentation is installed in the following directories:

  • /usr/webdocs/jMax on SGI platform
  • /usr/doc/jMax-$(version) on Linux platform
